# TypeScript コーディングルール

## 型

- `any` は使わない。型が不明な場合は `unknown` を使い、型ガードで絞り込む
- 型アサーション (`as`) より型ガード (`is`) を優先する
- ユーティリティ型（`Pick`, `Omit`, `Partial`）を活用して重複を避ける
- `enum` より union type を使う
- 外部APIのレスポンスは専用の型を定義し、zodなどでバリデーションする

## スタイル

- インデントは2スペース
- `const` をデフォルトにする。`let` は再代入が必要な場合のみ。`var` は禁止
- 早期リターンでネストを浅く保つ
- Optional chainingとnullish coalescingを活用する

## 関数

- 引数が3つ以上ならオブジェクト引数にする
- 副作用のない純粋関数を優先する
- async/awaitを使う。`.then()` チェーンは避ける
- コールバック地獄を避ける

## React（TSXファイル）

- コンポーネントは関数コンポーネントで書く
- Props型はコンポーネントと同じファイルに定義する
- カスタムフックで状態ロジックを分離する
- `useEffect` の依存配列は正確に保つ。ESLintの警告を無視しない

## 避けること

- `// @ts-ignore` や `// @ts-expect-error` の安易な使用
- 非nullアサーション (`!`) の乱用
- `object` 型の使用（具体的な型を定義する）
