# TypeScript 設計ルール

## モジュール設計

- 1ファイル1エクスポートを基本とする
- barrel export (`index.ts`) は共有モジュールのみで使う
- 循環参照が生じたら設計を見直す
- 型定義は使用箇所の近くに配置する。共有型のみ `types.ts` に分離

## ディレクトリ構成

- 機能単位（feature-based）でディレクトリを分ける
- テスト、型、ロジックは同じ機能ディレクトリにコロケーションする
- 共有UIコンポーネントは `components/` に配置
- 共有フックは `hooks/` に配置

## 状態管理（React）

- ローカルで済むなら `useState` / `useReducer` で十分
- Props drillingが3階層を超えたらContext / 状態管理ライブラリを検討する
- サーバー状態は専用ライブラリ（TanStack Query等）で管理する
- グローバル状態は最小限にする

## 依存関係

- 外部APIへのアクセスは専用のクライアントクラス/関数に集約する
- ビジネスロジックはUI層から分離する
- ライブラリ追加前に標準APIやネイティブ機能で解決できないか検討する
