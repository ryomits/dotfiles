# スキル作成スキル

新しいClaude Codeスキルを作成し、レビュー・自動修正まで一貫して行う。

## [1/3] スキル情報の取得

### 引数の解析

引数からスキル情報を取得する:

- `$ARGUMENTS` の最初の単語: スキル名
- `$ARGUMENTS` の残りの部分: スキルの説明

`$ARGUMENTS` が空の場合、AskUserQuestionで以下を順に質問する:
1. **スキル名**: ケバブケースで入力（例: pdf-converter, code-review, data-analyzer）
2. **スキルの説明**: 何を行うスキルか（例: PDFファイルを画像に変換し、OCR処理を行う）

### 追加情報の確認

以下をAskUserQuestionで確認する:

1. **スコープ**: グローバル（`~/.claude/skills/`）またはプロジェクト固有（`.claude/skills/`）
2. **引数**: スキルが受け取る引数があるか。あれば `argument-hint` を設定する
3. **手動実行のみか**: ユーザーが `/skill-name` で明示的に呼び出すときだけ動くか、Claudeが自動的に判断して呼び出してもよいか
4. **スキルの内容**: 何をするスキルか。ステップや条件があれば聞く

---

## [2/3] スキルの作成

### ディレクトリとSKILL.mdの作成

1. スキルのディレクトリを作成する
   - グローバル: `~/.claude/skills/<skill-name>/`
   - プロジェクト固有: `.claude/skills/<skill-name>/`
2. `SKILL.md` を作成する。以下のフォーマットに従う:

```markdown
---
name: <skill-name>
description: <説明>
argument-hint: "<引数のヒント>"  # 引数がある場合のみ
disable-model-invocation: true   # 手動実行のみの場合
---

# スキルの指示内容をここに記載
```

3. テンプレートや補助ファイルが必要であれば同じディレクトリに配置する

### 作成完了の確認

スキルが正常に作成されたことを確認する:

1. SKILL.mdファイルが生成されたか（Readツールで確認）
2. 必要なディレクトリ構造が作成されたか（Globツールで確認）

---

## [3/3] レビューと自動修正

### レビューの実行

`/review-skill` を呼び出して作成したスキルをレビューする。引数には作成したスキルのディレクトリパスを渡す。

### レビュー結果の処理

**問題がない場合**: 完了サマリーを表示して終了する。

**問題がある場合**:
1. 指摘された問題を自動的に修正する
2. 再度 `/review-skill` を実行して確認する
3. 問題がなくなるまで繰り返す（最大3回）
4. 3回試行しても解決しない場合はユーザーに手動対応を依頼する

### 自動修正のルール

- **構造的な問題**: ディレクトリ構造やファイル配置を修正
- **内容の問題**: SKILL.mdの内容を修正・改善
- **ベストプラクティス違反**: 推奨されるパターンに修正

---

## 完了サマリー

すべて完了したら以下の情報を表示する:

```
スキルの作成が完了しました

スキル名: [スキル名]
説明: [スキルの説明]

作成されたファイル:
- [ファイルパスのリスト]

レビュー結果:
- [問題なし / N件の問題を自動修正]
```

---

## 注意事項

- スキル名はケバブケース（例: `my-skill`）を推奨
- 既に同名のスキルが存在する場合は上書き前に確認する
- `.claude/commands/` は旧形式。`.claude/skills/` を使う

### エラーハンドリング

- スキル作成時のエラーは明確なエラーメッセージを表示する
- レビュー実行時のエラーはリトライオプションを提供する
- 修正不可能な問題はユーザーに報告する
